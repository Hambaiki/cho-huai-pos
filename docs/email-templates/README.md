# Email Templates

## Signup confirmation email

Use `signup-confirmation.html` as the **Confirm signup** template in Supabase Auth.

Note: this template is only used when email confirmation is enabled in Supabase Auth.

### Setup steps

1. Open Supabase dashboard for this project.
2. Go to **Authentication** > **Email Templates**.
3. Select **Confirm signup**.
4. Replace the existing HTML with the content from `docs/email-templates/signup-confirmation.html`.
5. Save and send a test email.

### Template variables used

- `{{ .ConfirmationURL }}`: the verification link generated by Supabase.
- `{{ .Email }}`: recipient email address.
- `{{ .Data.display_name }}`: user display name from signup form.
- `{{ .Data.app_name }}`: app display name passed by server action.
- `{{ .Data.support_email }}`: support email passed by server action.

### Optional environment variables

The signup actions now pass email template metadata from these server environment variables:

- `APP_NAME` (default: `Cho Huai POS`)
- `SUPPORT_EMAIL` (default: `support@example.com`)
- `EMAIL_CONFIRM_REDIRECT_URL` (optional)

If `EMAIL_CONFIRM_REDIRECT_URL` is set, Supabase will use it after email confirmation.
